  <action>
Create `src/app/[locale]/_components/Navbar.tsx` as a CLIENT component ("use client" directive).

**This component receives translations as props from its server parent (page.tsx in Plan 04).** It does NOT call getTranslations() itself.

Props type:
```typescript
type NavbarProps = {
  locale: "es" | "en";
  translations: {
    challenge: string;
    howItWorks: string;
    prizes: string;
    hubs: string;
    faq: string;
    sponsor: string;
    register: string;
  };
};
```

Implementation details (port from blueprint Navbar.tsx):

1. **Scroll-aware background:** Use `useScrollDirection()` from `@/hooks/useScrollDirection`. When scrollDirection is "down", apply `translateY(-100%)` to hide navbar. When "up" or null, show navbar. Also track scroll position (scrollY > 50) to switch from transparent to opaque background (`bg-background/80 backdrop-blur-lg border-b border-border`).

2. **Navigation links:** Array of nav link objects mapping translations to anchor targets:
   - challenge -> #manifesto
   - howItWorks -> #how-it-works
   - prizes -> #prizes
   - hubs -> #hubs
   - faq -> #faq
   - sponsor -> #sponsors
   Use plain `<a href="#section">` for anchor links (NOT next/link). Close mobile menu on click.

3. **Language switcher:** Two buttons (ES | EN) using `useRouter` and `usePathname` from `@/i18n/navigation`. On click, call `router.replace(pathname, { locale: newLocale, scroll: false })`. Highlight active locale with `font-bold text-foreground`, inactive with `text-muted-foreground`. Use the `font-mono-accent` utility class and a border container.

4. **Register CTA button** (desktop only in nav): `<a href="#register">` with primary button styling.

5. **Mobile menu:** Use `hamburger-react` package (Squash variant). State: `const [isOpen, setIsOpen] = useState(false)`.
   - `md:hidden` for hamburger trigger
   - `md:hidden` for dropdown panel
   - Panel renders all nav links + language switcher + register CTA
   - Close menu when any link is clicked
   - Style: `bg-background/95 backdrop-blur-lg border-b border-border`
   - Do NOT use AnimatePresence/motion for mobile panel animation (that's Phase 3 scope). Use simple conditional rendering.

6. **Logo:** Text placeholder "SpecHack" with `font-orbitron font-bold` + "by 404 Tech Found" subtitle in `font-mono text-[10px] text-muted-foreground` (visible `sm:block`, hidden on small mobile).

7. **Z-index:** Use `z-50` on navbar (matching blueprint).

8. **Styling:** Fixed position, full width. `transition-all duration-300` for show/hide animation. Container max width with `max-w-7xl mx-auto px-4 sm:px-6`.

Conventions: Tabs, double quotes, named export (`export function Navbar`), no default export, PascalCase file.
